BOM vs DOM

Le moteur JavaScript d’un navigateur est composé de 3 parties :

  • Le coeur est ce qui ne va pas différer peu importe l’hôte de JavaScript que l’on soit dans un hôte WebBrowser ou Acrobat ou encore dans Node
  • Le DOM est une API HTML au même titre des nouvelle que l’on vient de présenter. Le DOM n’est pas une spécificité JavaScript il a été implémenté dans de nombreux autres langages néanmoins au niveau du WebBrowser c’est la norme de ECMAScript et donc JS qui est utilisé
  • Le BOM est au niveau au dessus il permet l’interaction avec la fenêtre du navigateur pas le document en lui même.

JS Traversing II: The nodeName Property

The nodeName property specifies the name of a node.

  • nodeName is read-only
  • nodeName of an element node is the same as the tag name
  • nodeName of an attribute node is the attribute name
  • nodeName of a text node is always #text
  • nodeName of the document node is always #document

JS Traversing III: The nodeValue Property

The nodeValue property specifies the value of a node.

  • nodeValue for element nodes is undefined
  • nodeValue for text nodes is the text itself
  • nodeValue for attribute nodes is the attribute value
